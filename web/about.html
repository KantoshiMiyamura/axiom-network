Architect of Axiom Network

Kantoshi Miyamura

I build incorruptible money for a world where every adversary — classical, quantum, institutional — will eventually try to forge it.

Axiom Network is a Layer 1, proof-of-work blockchain written from scratch in Rust. Every signature is post-quantum (ML-DSA-87, FIPS 204). Every coin enters circulation through computational work. There is no pre-mine. There is no ICO. There is no team allocation. The chain is the product; everything else is noise.

Mission

Make sound money survive the next century.

Public-key cryptography built before 2024 will not survive a sufficiently large quantum computer. Most of the chains people hold value on today are signed with curves that will, eventually, be broken. Axiom is a refusal to wait for that day. Every transaction on this network is signed with a NIST Category 5 lattice scheme designed to resist both classical and quantum cryptanalysis. The chain is engineered to be running, intact, the morning after a cryptographically relevant quantum computer is announced.

Vision

A monetary network nobody can quietly debase.

A blockchain is a contract between strangers about what counts as a coin. If insiders can pre-mine, allocate themselves a treasury, or push consensus changes through a foundation, that contract is unenforceable. Axiom commits to the opposite: a fixed schedule, a single class of holders, and a protocol whose only legitimate path to change is a genuine fork that miners and node-runners adopt on their own machines. Sovereignty over money should belong to the people running the software.

Principles

The non-negotiables. Every line of code in this repository is in service of these.

01

Post-quantum from day one

ML-DSA-87 (FIPS 204) signatures on every transaction. SHA3-256 proof-of-work. Argon2id key derivation. XChaCha20-Poly1305 keystore encryption. No legacy ECDSA hiding anywhere in the trust path.

02

Fair distribution, no exceptions

Zero pre-mine. Zero ICO. Zero team allocation. Zero foundation treasury. Every AXM that exists was minted by the same proof-of-work coinbase rule that any miner can verify against the block they just produced.

03

Self-sovereign by default

You hold your keys. The wallet seals them with the operating system’s native key store. The cache is encrypted with a device-bound key. The clipboard is wiped after thirty seconds. The default RPC binds to localhost. Nothing in the stack assumes you trust a server.

04

Engineering discipline over marketing

Pinned Rust toolchain. Reproducible release builds. Signed tags and minisigned SHA256SUMS. Public CI. Documented audit policy with every advisory ignore justified. The release notes are an engineering log, not a press release.

05

Public testnet before mainnet

Mainnet is not live. The testnet is open so the protocol gets bruised in the open before any unit of value depends on it. Tokens on this testnet have no monetary value and never will. The chain you run today is the chain that earns the right to launch.

06

Open source, fully auditable

Every crate in the workspace is public. Consensus, networking, storage, signer, RPC — you can read the rule, find the test that pins it, and run the binary that enforces it. There is no closed component. There is no hosted service you have to trust.

On the name

A name is a liability. The protocol is the work; the author is incidental. The bust above is a reminder that the face matters less than the rules it ratified. Read the code, audit the consensus, run a node, and form your own judgment about whether this network earns your trust. Nothing else should.

Don’t take my word for it.

The protocol is the only authoritative statement. Read it, build it, run it, and break it.
