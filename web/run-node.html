Run a Node

Full nodes validate every transaction and block. Running one strengthens the network and gives you trustless access to the blockchain.

Configuration

Initialize creates the data directory and default configuration:

axiom init

Configuration file location:

  • Linux: ~/.axiom/axiom.conf
  • macOS: ~/Library/Application Support/axiom/axiom.conf
  • Windows: %APPDATA%\axiom\axiom.conf
Key configuration optionstoml
# Peer-to-peer network port
p2p_bind = "0.0.0.0:9000"

# Maximum peer connections
max_peers = 50

# Manual peers (Bitcoin-style — no auto-discovery, no DNS seeds).
# Add one or more friends/known nodes here, or pass --peer on the CLI.
peers = []

# RPC interface (127.0.0.1 = local only, safe default — never expose publicly)
rpc_bind = "127.0.0.1:8332"

# Enable mining (default: off)
mine = false
mining_threads = 0  # 0 = use all available cores

# Logging verbosity
log_level = "info"
🛡
RPC security

By default, RPC binds to 127.0.0.1 (localhost only). Do not expose RPC to the public internet without authentication and a reverse proxy.

Running the Node

Start the node
axiom start
Start with mining
axiom start --mine
Check status
axiom status

Running as a systemd Service (Linux)

/etc/systemd/system/axiom.serviceini
[Unit]
Description=Axiom Network Full Node
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=axiom
ExecStart=/home/axiom/.local/bin/axiom start
Restart=on-failure
RestartSec=10
LimitNOFILE=65535

[Install]
WantedBy=multi-user.target
Enable and start
sudo systemctl daemon-reload
sudo systemctl enable axiom
sudo systemctl start axiom
sudo journalctl -u axiom -f

Peer Discovery — Bitcoin-Style

Axiom has no project-operated bootstrap servers and no project-operated DNS seeds. A fresh node starts as a sovereign chain on its own; to join an existing network, you connect to peers you trust.

Start standalone — your node, your chain
axiom-node --p2p-bind 0.0.0.0:9000 --rpc-bind 127.0.0.1:8332
Join a friend's network — manual peer
axiom-node --p2p-bind 0.0.0.0:9000 --rpc-bind 127.0.0.1:8332 \
  --peer 1.2.3.4:9000 \
  --peer 5.6.7.8:9000

Once one peer is reached, the node learns about the rest of the network through peer-exchange (gossip). The first --peer is the only out-of-band step you need.

ℹ

If you are behind a NAT or firewall, forward TCP port 9000 to your node so other peers can dial in. Outbound-only nodes still work, but they don't help the network propagate blocks.

Best Practices

  • Keep your node software updated to the latest release
  • Use SSD storage for significantly faster sync and block validation
  • Run the node on a dedicated machine or VPS for maximum uptime
  • Monitor disk space — prune old logs periodically
  • Never expose RPC to the internet without proper authentication
  • Forward TCP port 9000 (P2P) to accept inbound peer connections
  • Synchronize your system clock with NTP
